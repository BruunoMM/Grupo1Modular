31 * outro módulo. Desta forma as alterações necessárias para cada
32 * módulo a testar concentram-se exclusivamente naquele módulo.

43 * =recuperar
44 * - Caso o comando imediatamente anterior ao comando
45 * =recuperar retorne TST_CondRetOK, obviemanete o comando
46 * executou corretamente, portanto o comando =recuperar
47 * reportará uma falha no script de teste, uma vez que não
48 * foi encontrada a falha esperada ao executar o comando.
49 * - Caso o comando antecessor retorne TST_CondRetErro,
50 * ou TST_CondRetParm (erro de parâmetros do comando)
51 * a falha reportada será desconsiderada, uma vez que
52 * era esperada. O comando =recuperar destina-se a
53 * apoiar o teste da corretude dos validadores de
54 * parâmetros dos comandos e das funções por eles
55 * ativadas.
56 * - caso o comando antecessor retorne outras condições, o
57 * comando =recuperar será tratado como falha.
58 *
59 * =bkpt - permite interagir com o depurador utilizado para
60 * desenvolver os módulos sob teste. O comando =bkpt
61 * executa um código "dummy" no qual pode ser inserido
62 * um "breakpoint" do depurador. Desta forma o teste pode
63 * prosseguir sem interrupção até o comando =bkpt.
64 * Para isto
65 * - apague ou desative todos os breakpoints do depurador
66 * - insira o comando =bkpt no script pouco antes do ponto
67 * de interrupção desejado
68 * - usando o depurador, estabeleça um breakpoint na linha de
69 * código: "Dummy = 0 ;" contida no módulo de teste
70 * genérico. Esta linha pode ser encontrada
71 * procurando por "BKPT_CMD"
72 * - dê partida no programa e espere até chegar ao breakpoint
73 * associado ao comando "dummy". Agora reative ou instale
74 * os breakpoints requeridos pela a sessão de depuração.
75 * e continue depurando
76 *
77 * =cancelar

304 * $FC Função: TSTG &Interpretar toda a massa de teste
305 *
306 * $ED Descrição da função
307 * Interpreta o arquivo de diretivas (script) de teste já aberto
308 *
309 * A função de teste é dividida em três classes de comandos:
310 *
311 * - os de teste genérico, que coordenam e controlam a realização do
312 * teste. Estes comandos são interpretados por esta função.
313 * - os de teste e controle de módulos componentes do arcabouço.
314 * Estes comandos são interpretados por funções de teste específicas
315 * para cada um dos módulos componentes.
316 * - os de teste específico do módulo (ou módulos) a ser testado.
317 * O módulo de teste específico interpreta os comandos projetados
318 * para realizar os testes do módulo a testar.
319 *
320 * A função de teste genérico lê um arquivo de diretivas de teste.
321 * Cada linha deste arquivo é um comando de teste.
322 * A função de teste lê uma a uma as linhas do arquivo, limpa os
323 * caracteres em branco no final da linha e procura interpretar o
324 * comando.
325 *
326 * As funções de teste específicas comunicam os resultados da execução
327 * de cada comando por meio de uma condição de retorno.
328 *
329 * Cada linha do arquivo script de teste corresponde a um comando de
330 * teste.
331 *
332 * Caso seja encontrada uma falha não esperada (ver comando =recuperar)
333 * os restantes comandos do caso de teste serão ignorados e contados
334 * como problemas.
335 *
336 * A sintaxe utilizada pelos comandos de teste é semelhante a assembler.
337 * Esta sintaxe facilita a análise:
338 *
339 * =<cmd> <parâmetros>
340 *
341 * em que:
342 * = - todos os comandos a serem interpretados iniciam
343 * com este caractere
344 * <cmd> - é o string que identifica o comando
345 * <parâmetros> - são zero ou mais itens a serem utilizados
346 * pelo comando de teste. Os itens devem, de preferência,
347 * ser separados um do outro por um caractere "espaço"
348 *
349 * Deve ser adotado o seguinte padrão de uso:
350 *
351 * - procure sempre utlizar uma das funções TST_CompararXXX, na forma:
352 * return TST_CompararXXX( valoresperado, funcao a testar , mensagem )
353 *
354 * - as funções de comparação fornecidas sempre geram as mensagens no
355 * padrão estipulado
356 *
357 * - quando não for possível utilizar uma função de comparação, reporte
358 * a falha, esperadas ou não, através da função
359 * TST_ImprimirPrefixo( Mensagem )
360 *
361 * - a seguir mostre o valor esperado e o obtido
362 *
363 * - Informações relativas a falhas dos testes são sinalizadas por
364 * uma mensagem iniciando com ">>>" o número da linha do comando de teste
365 * e o número de falhas encontradas até o momento.
366 *
367 * - Informações relativas à execução dos testes são sinalizadas por
368 * uma mensagem iniciando com "!!!".
369 *
370 * $FV Valor retornado
371 * TST_CondRetErro - caso tenha sido encontrado um ou mais problemas
372 * TST_CondRetOK - caso o teste tenha terminado com zero problemas
373 * não esperados.
374 *
375 * $FGP Tipos e Variáveis globais externas do próprio módulo
376 * pArqScript - deve referenciar o arquivo script aberto
377 * pArqLog - deve referenciar o arquivo log aberto
378 *
379 ***********************************************************************/
380 
381  TST_tpCondRet TST_RealizarTeste( char * Construto ) ;
